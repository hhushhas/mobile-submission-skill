The pipeline

A release operator, not a checklist.

Six phases carry an app from raw repository to submitted — with a final gate before anything ships.

01

Discover

Maps native config, permissions, assets, privacy URLs, and store metadata from the real repo.

02

Review

Runs readiness gates for blockers, high-risk issues, and unknowns before submission begins.

03

Prepare

Builds a structured packet, console task list, review notes, and secret references.

04

Execute

Delegates Apple and Google workstreams through CLI/API where possible, dry-run safe by default.

05

Guide

Turns dashboard-only forms into exact instructions — field names, control types, and values.

06

Submit

Final pre-submit gate, then TestFlight, App Review, or Play track submission when approved.

Safe by default

Designed to avoid accidental damage.

✓
Dry-run first

✓

✓
